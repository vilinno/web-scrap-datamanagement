# 山东大学课程实验报告：实验三 Web 数据的应用

## 实验题目

实验三：豆瓣读书数据分析与 Web 可视化应用。

## 实验目的

1. 对实验一采集的书籍数据进行清洗、统计和分析。
2. 使用 FastAPI、Jinja2、Pandas 和 ECharts 构建一个简单 Web 网站。
3. 通过图表展示评分分布、类型分布、出版年份趋势、热门出版社、高分书籍榜单等结果。

## 硬件环境

- 操作系统：Windows
- 运行目录：`F:\软件数据\系统文档\web-data-management-course-project`
- 浏览器：Chrome / Edge

## 软件环境

- 开发语言：Python、HTML、CSS、JavaScript
- 后端框架：FastAPI
- 模板引擎：Jinja2
- 数据处理：Pandas、SQLite
- 可视化模块：ECharts
- Web 源代码：`src/web`

## 实验步骤与内容

### 网站功能介绍

本网站是一个面向豆瓣读书数据的可视化仪表盘，主要功能包括：

1. 展示数据概览指标：记录数、平均评分、平均评价人数、出版社数量。
2. 支持按类型、最低评分、出版年份进行筛选。
3. 展示评分分布饼图。
4. 展示书籍类型分布柱状图。
5. 展示出版年份趋势折线图。
6. 展示热门出版社柱状图。
7. 展示评分与评价人数关系散点图。
8. 展示高分书籍榜单，并可点击跳转到源网页。

### 数据来源

- 来源网站：豆瓣读书
- 来源 URL：`https://book.douban.com/tag/小说?start=0&type=R`
- 数据文件：`data/processed/douban_books_clean.csv`
- 数据库：`data/processed/web_data_management.sqlite`

### 数据类型与数据字典

数据属于结构化数据，字段如下：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `title` | 文本 | 书名 |
| `author` | 文本 | 作者/译者 |
| `tag` | 分类文本 | 豆瓣标签 |
| `rating` | 数值 | 评分 |
| `rating_count` | 整数 | 评价人数 |
| `publisher` | 文本 | 出版社 |
| `publish_date` | 文本 | 出版日期 |
| `publish_year` | 整数 | 出版年份 |
| `price_number` | 数值 | 价格数字 |
| `summary` | 文本 | 简介 |
| `summary_length` | 整数 | 简介长度 |
| `detail_url` | 文本 | 详情页链接 |

### 数据量大小

- 清洗后记录数：620 条
- 2025 年及以后记录数：542 条
- 标签数量：5 类
- 出版社数量：123 个
- 有评分记录：605 条
- 平均评分：8.10
- 出版年份范围：2024 到 2026 年
- 清洗后 CSV 文件大小约 226 KB
- SQLite 数据库大小约 284 KB

### 可视化目标

| 图表名称 | 展示目的 | 对应数据 |
| --- | --- | --- |
| 评分分布 | 观察书籍评分主要集中在哪些区间 | `rating` |
| 类型分布 | 比较不同豆瓣标签的书籍数量 | `tag` |
| 出版年份趋势 | 观察采集数据的出版年份覆盖情况 | `publish_year` |
| 热门出版社 | 识别样本中出现频次较高的出版社 | `publisher` |
| 评分与评价人数关系 | 分析高评分是否伴随较高评价人数 | `rating`、`rating_count` |
| 高分书籍榜单 | 展示综合评分较高的书籍 | `title`、`author`、`rating` |

### Web 应用接口

| 接口 | 功能 |
| --- | --- |
| `/` | 渲染可视化首页 |
| `/api/summary` | 返回概览指标 |
| `/api/charts` | 返回图表数据 |
| `/api/books` | 返回高分书籍列表 |

## 结论分析与体会

本实验将实验一爬取的数据转化为一个可交互的 Web 数据应用。通过 Pandas 对原始字段进行清洗和派生，得到出版年份、价格数字、简介长度等适合分析的字段。FastAPI 提供页面和 JSON 接口，ECharts 负责前端图表渲染。与静态报告相比，Web 应用可以通过筛选条件动态观察不同类型、不同年份和不同评分区间下的数据变化，更直观地展示 Web 数据管理中“采集、清洗、存储、分析、展示”的完整流程。

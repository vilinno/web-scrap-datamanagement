# 山东大学课程实验报告：实验三 Web 数据的应用

## 实验题目

实验三：豆瓣读书数据分析与 Web 可视化应用。

## 实验目的

1. 对实验一采集的书籍数据进行清洗、统计和分析。
2. 使用 FastAPI、Jinja2、Pandas 和 ECharts 构建一个简单 Web 网站。
3. 通过图表展示评分分布、类型分布、出版年份趋势、热门出版社、高分书籍榜单等结果。

## 实验步骤与内容

### 网站功能介绍

本网站是一个面向豆瓣读书数据的可视化仪表盘，主要功能包括：

1. 展示数据概览指标：记录数、平均评分、平均评价人数、出版社数量。
2. 支持按类型、最低评分、出版年份进行筛选。
3. 展示评分分布饼图。
4. 展示书籍类型分布柱状图。
5. 展示出版年份趋势折线图。
6. 展示热门出版社柱状图。
7. 展示评分与评价人数关系散点图。
8. 展示高分书籍榜单，并可点击跳转到源网页。

### Web 应用架构与实现手段

本系统采用轻量级前后端分层架构：

1. 数据层：实验一爬取得到 `data/raw/douban_books_raw.csv`，再由 `src/data/clean_books.py` 清洗生成 `data/processed/douban_books_clean.csv` 和 `data/processed/web_data_management.sqlite`。
2. 后端层：`src/web/main.py` 使用 FastAPI 搭建 Web 服务，使用 Pandas 读取 CSV 数据，并提供 `/api/summary`、`/api/charts`、`/api/books` 三个 JSON 接口。
3. 模板层：Jinja2 渲染 `src/web/templates/index.html`，在页面初始化时注入类型和年份筛选项。
4. 前端层：`src/web/static/app.js` 使用浏览器 `fetch` 请求后端接口，使用 ECharts 绘制评分分布、类型分布、年份趋势、出版社排名和散点图。
5. 样式层：`src/web/static/styles.css` 实现响应式布局，桌面端为双列图表，移动端自动折叠为单列。

这种架构的优点是结构清晰、启动简单、便于课程演示：数据处理、接口服务和图表展示相互独立，但都围绕同一份清洗后的数据工作。

### 运行命令

如果已经存在 `data/processed/douban_books_clean.csv`，可以直接启动 Web 服务：

```powershell
python -m uvicorn src.web.main:app --reload --host 127.0.0.1 --port 8000
```

浏览器访问：`http://127.0.0.1:8000`

如果需要从头重新生成数据，先运行：

```powershell
python src/spiders/douban_books.py --target 620 --pages 8 --sort-type R
python src/data/clean_books.py
```

## 结论分析与体会

本实验将实验一爬取的数据转化为一个可交互的 Web 数据应用。通过 Pandas 对原始字段进行清洗和派生，得到出版年份、价格数字、简介长度等适合分析的字段。FastAPI 提供页面和 JSON 接口，ECharts 负责前端图表渲染。与静态报告相比，Web 应用可以通过筛选条件动态观察不同类型、不同年份和不同评分区间下的数据变化，更直观地展示 Web 数据管理中“采集、清洗、存储、分析、展示”的完整流程。

# 山东大学课程实验报告：实验一 简单爬虫

## 实验题目

实验一：简单爬虫，豆瓣读书最近出版书籍数据采集。

## 实验目的

1. 选择 Python 作为开发语言，掌握 Requests、BeautifulSoup、CSV、SQLite 等爬虫与数据保存工具。
2. 分析豆瓣读书标签页的网页结构，定位书籍列表中的标题、作者、标签、评分、出版社、出版时间、价格等字段。
3. 编写可重复运行的爬虫程序，采集规模较大、字段较丰富的数据，为实验三的数据分析和 Web 可视化提供基础。

## 实验步骤与内容

### 目标网站分析

- 网站名称：豆瓣读书
- URL：`https://book.douban.com/tag/小说?start=0&type=R`
- 网页类型：HTML 静态列表页为主，列表页中包含书籍卡片。
- 数据区域：每本书位于 `li.subject-item` 节点中，标题在 `h2 a`，出版信息在 `.pub`，评分在 `.rating_nums`，评价人数在 `.pl`，简介在 `p`。
- 排序参数：使用 `type=R`，优先获取出版时间较新的书籍，更贴合“最近一年出版书籍”的要求。

### 爬取目标字段

本次数据字段超过 4 个，便于后续多角度分析：

| 字段 | 含义 |
| --- | --- |
| `title` | 书名 |
| `author` | 作者/译者 |
| `tag` | 豆瓣标签类型 |
| `rating` | 评分 |
| `rating_count` | 评价人数 |
| `publisher` | 出版社 |
| `publish_date` | 出版日期 |
| `price` | 价格 |
| `summary` | 简介 |
| `detail_url` | 详情页地址 |
| `cover_url` | 封面地址 |

### 工作思路

1. 构造豆瓣标签页 URL，按标签和分页遍历。
2. 使用浏览器常见请求头访问页面，降低被误判为异常访问的概率。
3. 使用 BeautifulSoup 解析列表节点，拆分 `.pub` 字段得到作者、出版社、出版日期和价格。
4. 使用集合记录详情页 URL，避免跨标签重复数据。
5. 将原始数据保存为 CSV 和 SQLite，方便后续清洗与分析。

### 源代码分析

核心代码位于 `DoubanBookSpider.crawl()`：

- 根据标签和页码拼接 `https://book.douban.com/tag/{tag}?start={start}&type=R`。
- `fetch()` 负责请求、重试和验证页检测。
- `parse_list_page()` 负责解析书籍列表节点。
- `save_csv()` 和 `save_sqlite()` 分别保存文本文件和数据库文件。

`parse_pub_line()` 用于拆分豆瓣出版信息行。由于作者字段中可能包含译者、校注者等多个斜杠，代码从末尾向前识别价格、出版日期、出版社，剩余部分合并为作者信息。

## 结论分析与体会

本实验完成了豆瓣读书数据采集，最终获得 620 条结构化书籍数据，字段包含文本、数值、日期、链接等多种类型。实验中发现豆瓣列表页数据结构较稳定，但请求过快可能出现超时或验证风险，因此爬虫加入了随机延迟、重试和异常提示。通过 CSV 与 SQLite 两种格式保存数据，既便于人工查看，也便于实验三的 Web 应用读取。

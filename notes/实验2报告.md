# 山东大学课程实验报告：实验二 克服反爬虫障碍

## 实验题目

实验二：克服动态加载 / AJAX 反爬障碍，采集 B 站排行榜数据。

## 实验目的

1. 选择具有反爬虫障碍的网站，分析普通 HTML 解析无法直接获取完整数据的原因。
2. 使用浏览器开发者工具定位真实 AJAX 接口。
3. 构造请求头、Referer 和 Cookie，编写代码获取 JSON 数据并保存。

- 运行目录：`F:\软件数据\系统文档\web-data-management-course-project`
- 网络环境：普通校园/家庭网络

- 开发语言：Python 3
- 开发工具：PowerShell / 浏览器开发者工具

## 实验步骤与内容

### 目标网站分析

- 网站名称：哔哩哔哩排行榜
- 页面 URL：`https://www.bilibili.com/v/popular/rank/all`
- 数据接口：`https://api.bilibili.com/x/web-interface/ranking/v2?rid=0&type=all`
- 反爬类型：动态加载、AJAX 接口、请求头校验、Cookie 校验、Python 请求可能触发风控码。

### 反爬障碍展示

直接使用 Python `requests` 访问接口时，曾出现如下异常证据：

```text
requests 返回异常：{'code': -352, 'message': '-352', 'ttl': 1}
请求地址：https://api.bilibili.com/x/web-interface/ranking/v2?rid=0&type=all
```

证据文件保存于：`data/raw/bilibili_request_evidence.txt`。

这说明目标接口不是简单裸请求即可稳定访问，需要模拟浏览器访问过程。

### 工作思路

1. 打开 B 站排行榜页面，使用浏览器开发者工具的 Network 面板筛选 XHR/Fetch 请求。
2. 定位 `ranking/v2` JSON 接口，确认返回字段包括视频标题、UP 主、分区、播放量、弹幕数、评论数、收藏数、投币数、分享数、点赞数等。
3. 在脚本中先使用 `requests` 携带浏览器请求头访问接口。
4. 如果出现 SSL 断连或 `-352` 风控码，则先访问 B 站首页获取 Cookie，再带 Cookie 访问排行榜接口。
5. 将 JSON 数据转换为结构化 CSV、JSON 和 SQLite。

## 结论分析与体会

本实验体现了动态网页和普通静态网页的差异。B 站排行榜页面中的可视数据并不完全写在初始 HTML 中，而是由浏览器后续请求 JSON 接口得到。直接用 Python 请求接口可能触发风控码 `-352`，说明目标站点会结合请求头、Cookie 或客户端特征判断访问来源。通过先获取首页 Cookie，再携带 Referer、Origin、User-Agent 等浏览器请求头访问接口，可以稳定获得排行榜数据。本实验没有绕过登录、验证码或付费权限，只采集公开排行榜数据，符合课程实验和合理访问原则。
